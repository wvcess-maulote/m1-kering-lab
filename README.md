# m1-kering-lab

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面，内容涵盖各类实验性、展示性及辅助功能的静态页面。这些页面彼此独立，不依赖于特定域名或网站，可作为轻量级的 Web 资源进行访问和引用。

## 目录说明

```
m1-kering-lab/
├── pages/          # 存放所有独立 HTML 页面
├── assets/         # 页面所需的静态资源（CSS、JS、图片等）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名通常反映其功能或主题。
- `assets/`：按页面或类别组织的资源文件，部分页面可能内嵌资源而不依赖外部文件。

## 页面归档说明

- 所有页面均为纯前端实现，无后端依赖。
- 页面内容可能包含演示、工具、信息展示等，无特定商业或推广目的。
- 部分页面可能处于实验或未完成状态，功能与样式可能存在变动。
- 归档的页面不保证长期可用性或兼容性，请以实际运行效果为准。

## 维护说明

- 本仓库不定期更新，新增或修改页面均通过 Pull Request 或直接提交完成。
- 欢迎提交 Issue 反馈页面异常或提出改进建议。
- 不接受涉及具体域名、商业推广或违反平台政策的页面提交。
- 仓库维护者保留对页面内容进行修改或移除的权利。

## 使用方式

可通过 GitHub Pages 或其他静态托管服务直接访问 `pages/` 目录下的 HTML 文件。如需本地查看，请克隆仓库后使用浏览器打开对应文件。

```bash
git clone https://github.com/your-username/m1-kering-lab.git
cd m1-kering-lab/pages
# 打开任意 HTML 文件
```
